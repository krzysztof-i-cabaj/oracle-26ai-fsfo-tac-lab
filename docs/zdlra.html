Knowledge base

5 new lessons learned (#30 → #34)

Discovered during the autonomous test session. Cumulative catalog now at 34 lessons in docs/10_Troubleshooting.md.

#30 RAC PDB PITR requires CLOSE IMMEDIATE INSTANCES=ALL. Without INSTANCES=ALL RMAN throws ORA-65025: Pluggable database is not closed on all instances. Single-instance docs assumed CLOSE IMMEDIATE alone is enough.
#31 SQL*Plus heredoc over SSH is session-isolated. ALTER SESSION SET CONTAINER in one heredoc does NOT persist to the next. Workaround: use sqlplus -S / as sysdba @/tmp/script.sql with the entire sequence in one SQL file.
#32 SET FEEDBACK ON + ECHO OFF as must-have. First line of every SQL file: SET LINESIZE 220 PAGESIZE 50 FEEDBACK ON HEADING ON ECHO OFF. Defaults may hide query results, looking like silent failures.
#33 Table "vanished" between SQL sessions — created + committed, gone before next phase, not in dba_recyclebin. Likely related to #31 session isolation. Workaround: fresh CREATE in same session as test. Root cause not yet diagnosed.
#34 Block corruption demo (B-5) unavailable for ASM datafiles. doc 08 §B-5 assumed filesystem datafiles (dd if=/dev/zero of=/u02/oradata/...). LAB datafiles on +DATA/PRIM/... require ASMCMD-based or DBMS_REPAIR approach.
