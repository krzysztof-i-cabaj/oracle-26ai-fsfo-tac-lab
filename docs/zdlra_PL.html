Knowledge base

5 nowych lekcji (#30 → #34)

Odkryte podczas autonomicznej sesji testowej. Kumulatywny katalog ma teraz 34 lekcje w docs/10_Troubleshooting_PL.md.

#30 RAC PDB PITR wymaga CLOSE IMMEDIATE INSTANCES=ALL. Bez INSTANCES=ALL RMAN rzuca ORA-65025: Pluggable database is not closed on all instances. Single-instance docs zakładały że samo CLOSE IMMEDIATE wystarczy.
#31 SQL*Plus heredoc przez SSH jest session-isolated. ALTER SESSION SET CONTAINER w jednym heredoc NIE persists do następnego. Workaround: użyj sqlplus -S / as sysdba @/tmp/script.sql z całą sekwencją w jednym SQL pliku.
#32 SET FEEDBACK ON + ECHO OFF jako must-have. Pierwsza linia każdego SQL pliku: SET LINESIZE 220 PAGESIZE 50 FEEDBACK ON HEADING ON ECHO OFF. Domyślne ustawienia mogą ukrywać wyniki SELECT, wyglądając jak silent failures.
#33 Tabela "zniknęła" między SQL sessions — utworzona + zacommitowana, brak przed kolejną fazą, nie w dba_recyclebin. Prawdopodobnie związane z #31 izolacją sesji. Workaround: świeży CREATE w tej samej sesji co test. Root cause nie zdiagnozowany.
#34 Block corruption demo (B-5) niedostępne dla ASM datafiles. doc 08 §B-5 zakładał filesystem datafiles (dd if=/dev/zero of=/u02/oradata/...). LAB datafiles na +DATA/PRIM/... wymagają ASMCMD-based lub DBMS_REPAIR approach.
