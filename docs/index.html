Key features

Four pillars of high availability

Every MAA tier implemented and tested in a single laboratory.

⚡

Fast-Start Failover (FSFO)

Automatic switch without DBA intervention. A 3-Observer quorum (Master obs_ext on infra01 + Backup obs_dc + obs_dr) eliminates false positives and delivers HA even when one site fails.

♻️

Transparent Application Continuity

Java UCP sessions are restored automatically after failover — the application sees no error. The TestHarness.java client uses OracleDataSourceImpl with full TAC replay and Transaction Guard.

📖

Active Data Guard — always READ ONLY

Standby permanently in READ ONLY WITH APPLY mode. Oracle Restart + SAVE STATE guarantees that after reboot or any switchover the database opens without manual intervention.

🔗

Real Application Clusters (RAC)

2-node cluster on shared iSCSI storage (ASM disk groups: +DATA, +RECO, +OCR). 9 GB RAM per node — meets cluvfy requirements for Oracle 23.26.

🛡️

Oracle Restart on Standby

The MYAPP_TAC service is registered with -role PRIMARY. After FSFO failover CRS automatically resumes the service — RTO drops to 5–45 s without extra scripting.

🤖

Full automation

15 steps from empty VMs to running TAC tests. OS kickstart, silent GI/DB install, DG Broker, FSFO, Observers, ONS — all driven by bash scripts and .rsp files. Idempotent and repeatable.

Step-by-step installation

From empty VMs to working MAA

A linear, repeatable path with full automation at every stage.

1

Create VMs in VirtualBox

PowerShell vbox_create_vms.ps1 — 5 VMs with paravirt KVM, virtio-net, hostiocache selectively

2

Kickstart OS Install

Oracle Linux 8.10 unattended — HTTP server + ks-*.cfg files with HugePages, THP off, fstab

3

Pre-flight validation

validate_env.sh --full on prim01 and stby01 — DNS, NTP, SSH, ASM disks, HugePages, THP, memlock

4

iSCSI Target + Initiator

LVM block backstore on infra01 (DATA/RECO/OCR LUNs), initiator on prim01/prim02

5

SSH Equivalency

Full mesh oracle/grid: prim01 ↔ prim02 ↔ stby01 ↔ infra01 — via ssh_setup.sh

6

Grid Infrastructure (RAC)

Silent GI 23.26 install on prim01+prim02 — gridSetup.sh, root.sh sequentially

7

Oracle Restart (Standby)

GI Standalone on stby01 — manages the MYAPP_TAC service with -role PRIMARY after failover

8

DB Software Install

Software-only on prim01 and stby01 — runInstaller with db.rsp, no database creation

9

Create the Primary RAC

DBCA silent — CDB PRIM + PDB APPPDB, large pages only, lock_sga, Flashback ON

10

Active Data Guard + Broker

CREATE PHYSICAL STANDBY through Broker, STBY permanently READ ONLY WITH APPLY + SAVE STATE

11

FSFO Master Observer

obs_ext on infra01 — auto-login SSO wallet, MaxAvailability mode, SYNC redo

12

Backup Observers

obs_dc (prim01) + obs_dr (stby01) — FSFO quorum survives Master Observer failure

13

TAC Service + Cross-site ONS

MYAPP_TAC service, FAN events, cross-site ons.config, systemd oracle-ons.service

14

Java TestHarness client

Compile and run TestHarness.java — UCP pool, TAC replay, session metrics

15

Test scenarios

Planned switchover, FSFO failover, TAC replay under load, apply lag, Master Observer outage
