5 bugów które wyszły dopiero w eksploatacji

Te luki nie ujawniłyby się przy pojedynczym uruchomieniu — pojawiły się dopiero przy wielokrotnych role-swap'ach + faktycznym obciążeniu.

S28-64

APPPDB OPEN MODE persistence po switchover

Problem

SAVE STATE zachowuje stan PDB per-rola w danym momencie. Po switchover stary stan persyste, APPPDB pozostaje w MOUNTED.

Objaw

ORA-65054 przy próbie connect do APPPDB po przejęciu roli primary przez stby01.

Fix

srvctl modify pdb -policy AUTOMATIC -role PRIMARY na obu klastrach — CRS-managed auto-OPEN po role swap.

S28-65

SCAN listener BLOCKED service po wielokrotnych role swap

Problem

Po 3+ role swap'ach service MYAPP_TAC w stanie BLOCKED na SCAN listener. JDBC widział "ORA-12514 listener does not currently know of service".

Objaw

TestHarness_TAC łączył się do stby01 zamiast prim01 (service zarejestrowany na obu). Routing nie szanował -role PRIMARY.

Fix

srvctl stop service ... ; srvctl start service ... czyste re-registration. Fix dla Oracle Restart na SI.

S28-66

DGMGRL syntax change w 26ai dla Master Observer promotion

Problem

Legacy 19c EDIT CONFIGURATION SET MASTEROBSERVER='<name>' nie działa w 26ai — syntax error.

Objaw

"DGM-XXXX: invalid syntax" przy próbie ręcznej promocji backup observera w trakcie scenariusza 5.

Fix

W 26ai nowy syntax top-level: SET MASTEROBSERVER TO <name>;. Skrypty deployment zaktualizowane.

S28-67

FastStartFailoverLagLimit chroni Transport, NIE Apply

Problem

Założenie: "LagLimit=0 blokuje failover gdy standby ma jakikolwiek lag". Założenie BŁĘDNE.

Objaw

W scenariuszu 4 apply był stoppowany, apply lag urósł do 5 min — failover i tak zadziałał, bo Transport Lag był = 0 (redo dotarł do SRL).

Fix

Lekcja edukacyjna (niefizyczny fix). MaxAvailability + AFFIRM = "redo dotarł" ≠ "redo zaaplikowany". Jeśli chcesz blokować przy Apply Lag — manual STARTUP MOUNT po failover (broker auto-reinstate).

S28-68

systemd RestartSec=10s blokuje testy outage observera

Problem

W scenariuszu 5 chcieliśmy zademonstrować auto-promocję backup observera. kill -9 Master nie pomagał — systemd restartował proces zanim broker zdążył wykryć stratę.

Objaw

Auto-promote nie wystąpił, bo Master Observer wracał szybciej niż FastStartFailoverThreshold=30s.

Fix

Workaround: explicit SET MASTEROBSERVER TO obs_dc dla demo. Production: rozważyć większy RestartSec dla unit dgmgrl-observer-ext, ale tylko jeśli outage demo jest priorytetem.
